1.3 starter pom 的依赖支持

Spring Boot 自身提供了很多功能的自动配置支持。只要在 pom.xml 中引入对应的 starter pom 即可使用在项目依赖中找到对应功能支持的Bean。

除了官方的 starter pom 之外，还有第三方为 Spring Boot 所写的 starter pom，也是在 pom.xml 中引入相应依赖即可。 – 比如 Spring Batch 批处理sql

2. 外部配置

Spring Boot 允许使用 properties 或 yaml 文件或者命令行参数的方式，进行自定义外部配置。

  1. 使用命令行参数的方式
    使用默认配置运行 springboot打包后的jar包

    使用命令行参数的方式，在运行时修改Tomcat端口号

  2. 使用@Value方式，注入 properties 文件中的值 – 可以用来设置一些静态资源，文件缓存路径等。

注意：这种方式没有指定资源文件位置的话，只能在 application.yml 中编写自定义的值。

  1. 通过 Bean 类型映射一个properties的属性（基于properties）

通过 @ConfigurationProperties注解，使用类型安全的配置方式。

Spring Boot 1.4之后取消了 ConfigurationProperties 的 locations 属性，无法指定属性资源的位置。

两种替代方案：

  1. 使用 @Component 注册为组件，然后使用 @PropertySource 指定资源位置。

缺点：只能使用 xxx.properties 文件。

  1. 新建 application-animal.yml 文件，然后在 application.yml 中开启该属性文件

缺点：要按 SpringBoot 的规定的格式配置，即 application-XXX.yml
一次只能开启一个 -xxx.yml 文件，最好用来配置环境变量。
属于 Profile 配置内容。

3. 日志配置

默认使用 Logback 作为日志框架，支持Log4J,Log4J2,Java Util Logging, Logback等作为日志框架。

4. Profile 配置

需要注意配置文件的文件命名格式应该为：application-{xxx}.yml 才可以被 springboot 自动解析。

一次只能开启一个 -xxx.yml 文件，最好用来配置环境变量。
