Spring 高级话题

3.1 Spring Aware

Spring Aware 能够让自定义的Bean获得Spring容器的服务。因为 Spring容器（ApplicationContext） 接口继承了 MessageSource 接口、ApplicationEventPublisher接口和ResourceLoader接口。

Spring Aware 是设计也用来 Spring 框架内部使用的，如果在自定义Bean中使用了Spring Aware（即实现了相关的 Aware接口），那么这个Bean就和Spring的框架耦合。

Spring Aware 提供的相关 Aware 接口如下：可以按需implement对应的接口，调用相关功能。
BeanNameAware 获取容器中Bean的名称
BeanFactoryAware 获取当前 bean factory，这样可以调用容器服务
ApplicationContextAware 当前的 application context，这样可以调用容器的服务
MessageSourceAware 获取 message source，可以获取文本信息
ApplicationEventPublisherAware 应用事件发布其，可以发布事件，02-spring常用配置的事件Event也可以通过实现这个接口来发布事件
ResourceLoaderAware 获取资源加载器，可以获得外部资源文件

3.6 @Enable* 注解的工作原理

通过 @Enable* 可以开启一项功能的支持。减少自己配置大量代码，降低使用难度。

注解名称 注解含义 出现位置
@EnableAspectJAutoProxy 开启对 AspectJ 自动代理的支持 part 1
@EnableAsync 开启异步方法的支持 part 1
@EnableScheduling 开启计划任务的支持 part 1
@EnableWebMvc Web MVC 的配置支持 part 2
@EnableConfigurationProperties 开启对 ConfigurationProperties 注解配置 Bean 的支持 part 3
@EnableJpaRepositories 开启对 Spring Data JPS Repository 的支持 part 3
@EnableTransactionManagement 注解式事务的支持 part 3
@EnableCaching 注解式缓存的支持 part 3

在 @Enable*注解的源码中，所有注解都有一个 @Import注解，用来导入配置类。

  1. 直接导入配置类
  2. 依据条件按选择配置类
  3. 动态注册 Bean

3.7 Spring 集成测试

Spring TestContext Framework 对集成测试提供顶级支持，不依赖于特定的测试框架：可以使用 Junit，也可以使用 TestNG。
集成测试涉及程序中的各个分层，前面“运行”部分都可以用 Spring 测试来检验功能是否正常。

基于 Maven 构建的项目结构默认有关的测试目录为：src/test/java (测试代码) 和 src/test/resource（测试资源） 区别于 src/main/java（项目源码） 和 src/main/resource（项目资源）

Spring 提供了 SrpingJUnit4ClassRunner 类，用于提供 Spring TestContext Framework 的功能。配合 @ContextConfiguration(classes = {xxxConfig.class})来配置 Application Context，通过 @ActiveProfiles("xxx")来确定活动的 profile

src\mian 和 src\test 下 com.wisely.highlight_spring4.ch3.fortest 包的内容。 – 注意：spring-test 的版本要和本身 Spring 的版本一致，否则运行报错。
