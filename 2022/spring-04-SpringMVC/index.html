4.2 Spring MVC 常用注解

@Controller

@Controller注解在类上。
用来声明控制器Bean, Dispatcher Servlet 会自动扫描该注解下的类。
普通Bean时，使用 @Component、@Service、@Repository、@Controller都是等同的。

@RequestMapping

注解在类或者方法之上。
用来映射 Web 请求（访问路径和参数）、处理类和方法。
注解在方法上的 @RequestMapping 路径会继承继承在类上的路径。
支持 Servlet 的request 和 response作为参数，也支持对 request和response的媒体类型进行配置（文件格式）

@ResponseBody

注解在返回值前面或者方法上。
支持将返回值放在 response 体内，而不是返回一个页面。
在基于 Ajax的程序时，可以以此注解返回数据而不是也页面。

@RequestBody

注解在 参数前。
允许 request 的参数在 request 体中，而不是直接拼接在链接地址中。

@PathVariable

注解在 参数前。
用来接收路径参数，如 /news/001, 可以接收 001 作为参数

@RestController

注解在 类。
一个组合注解，组合了 @Controller 和 @ResponseBody。
当只开发一个与页面交互数据的控制时，可以使用此注解。

3. ControllerAdvice

类上：
@ControllerAdvice: 作用是将对控制器的全局配置放在同一个位置
方法内：
@ExceptionHandler(value=Exception.class): 全局处理控制器中的异常，可以通过value来过滤拦截天降
@ModelAttribute: 将键值对添加到全局，所有注解了 @RequeMapping()的方法，都可以通过@ModelAttribute(“xxx”)获取对用值。
