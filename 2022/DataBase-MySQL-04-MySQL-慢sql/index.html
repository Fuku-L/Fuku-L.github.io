事务隔离性的实现

通过 MVCC 实现：

可重复读的隔离级别下，在事务启动时，将整个数据库的状态创建“视图” – 通过一个数组记录当前所有未提交的事务id，将最高的事务 id+1 作为当前事务id。

通过该数组可以将读到的 row_trx_id 分为以下三个部分：

在“可重复读”隔离级别下，读到数据库中某行记录的 row_trx_id 判断落在上图的那块区域：

  1. row_trx_id < 低水位id，该行此版本在绿色区，可见。
  2. row_trx_id > 高水位id，该行此版本在红色区，不可见。
  3. 低水位id < row_trx_id < 高水位id，在黄色区域，分为两种情况：
    a. 在数组中找得到 row_trx_id，说明该行数据事务未提交，不可见。
    b. 在数据中找不到，说明该行数据事务已提交，可见。

如果读取到的行版本不可见，借助 undo log 将数据回滚至满足要求的版本生成对应的“快照数据”。

小结：判断一条数据在某个事务中可能存在的三种情况：

  1. 版本未提交，不可见。
  2. 版本已提交，但是是在当前事务创建之后提交的，不可见。
  3. 版本已提交，在事务创建之前提交的，可见。
