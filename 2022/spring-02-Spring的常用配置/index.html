Spring 的常用配置

2.1 Bean的Scope

Spring 中 Bean的Scope有5种:分别是 Singleton,Prototype,Request,Session,GlobalSession
另外,在 Spring Batch 中还有一个 Scope 是使用 @StepScope.

@Bean的注解默认是’Singleton’单例模式,在容器中只会存在一个.

2.2 EL-Spring 表达式语言

在 Spring 中涉及调用各种资源的情况,包括普通文件,网址,配置文件,系统环境变量等,可以使用 EL表达式语言实现资源的注入.

有7种方式：

  1. 注入普通字符串
  2. 注入操作系统属性
  3. 注入表达式结果
  4. 其他Bean属性
  5. 文件资源
  6. 网址资源
  7. 配置文件

具体实例代码查看:

  1. com.wisely.highlight_spring4.ch2.el 包
  2. Resource目录下的ch2\el文件。
  3. pom.xml 导入 commons-io 提供文件流读写的支持

2.3 Bean 的初始化和销毁

Spring 对 Bean 的生命周期的操作提供了支持。在使用 Java 配置和注解配置下提供了如下两种方式：
（1） Java 配置方式：使用@Bean的initMethod和destroyMethod（相当于 xml 配置的 init-method 和 destroy-method）
（2）注解方式：利用JSR-250的@PostConstruct 和 @PreDestroy。

具体实例代码查看:
com.wisely.highlight_spring4.ch2.prepost 包以及需要在 pom.xml 中导入 JSR250-api 提供注解方式的支持。

2.5 Application Event（事件）

Spring 的事件为Bean与Bean之间的消息通信提供了支持。

当一个Bean A处理完一个任务之后，希望另一个Bean B直到并能做出相应处理。这就需要 Bean B 监听当前Bean A 所发送的事件Application Event。

Spring 的事件需要遵循如下流程：

  1. 自定义事件，集成 ApplicationEvent
  2. 定义事件监听器，在接收消息者B中，实现 ApplicationListener
  3. 发布事件，在发送消息者A中，注入容器并使用容器发布事件

具体实例代码查看:
com.wisely.highlight_spring4.ch2.event 包 或者 com.wisely.highlight_spring4.ch2.AtoB_Event 包中。
