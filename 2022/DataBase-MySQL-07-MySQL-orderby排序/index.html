rowid 排序

当待排序的数据行长度超过 max_length_for_sort_data 时，MySQL 认为单行过长，需要换成row_id排序。

rowid 排序流程

rowid 排序需要多访问一次表t的主键索引。获取需要的结果集数据返回给客户端。（一条一条直接返回，不需要额外的内存空间存储）

MySQL 排序算法的选择

MySQL 设计思想：如果内存够，就要多利用内存，尽量减少磁盘访问。

对于InnoDB表来说，rowid排序会要求回表多造成磁盘读，因此不被优先选择。

因此，只有在排序内存过小的情况下，才会选择rowid排序算法。否则会优先选择“全字段排序”，将结果集中的字段放在 sort_buffer 中。
在内存中对 sort_buffer 排序完成之后直接从返回查询结果，不需要再进行回表查询。

不是所有 order by 语句，都需要额外的排序。通过索引覆盖可以避免额外的排序操作。
